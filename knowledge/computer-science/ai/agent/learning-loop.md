## 核心 Loop（6 步循环）

```
            ┌──────────────┐
            │ 📥 Capture   │  每天 < 5 分钟
            │  收新东西    │  inbox/YYYY-MM-DD.md
            └──────┬───────┘
                   │
                   ▼
            ┌──────────────┐
            │ 🔍 Triage    │  周末 30-60 分钟
            │  过滤 ⭐📖🗑 │
            └──────┬───────┘
                   │ (⭐ 项)
                   ▼
            ┌──────────────┐
            │ 📝 Deep Dive │  写卡片或笔记
            │  弄透一个概念 │
            └──────┬───────┘
                   │
                   ▼
            ┌──────────────┐
            │ 🔗 Connect   │  放进 framework 正确层
            │  标主层+跨层  │
            └──────┬───────┘
                   │
                   ▼
            ┌──────────────┐
            │ 🗣️ Teach     │  讲给 Ori 听
            │  苏格拉底追问 │
            └──────┬───────┘
                   │
                   ▼
            ┌──────────────┐
            │ 📊 Audit     │  季度：framework 整体复盘
            │  每层打分     │
            └──────┬───────┘
                   │
                   └──────► 回到 Capture（loop 闭合）
```

## 每步详细

### 📥 Capture（每天 < 5 分钟）
- 看到 agent 相关推文/文章/视频 → 丢进 `inbox/YYYY-MM-DD.md`
- 格式：`- [标题](链接) — 一句话摘要`
- **不要整理、不要思考、不要读全文** — 只收集

### 🔍 Triage（周末 30-60 分钟）
- ⭐ 必学 → 转 `cards/<concept>.md` 或放进对应层文件夹
- 📖 感兴趣 → 排进下周
- 🗑 跳过 → 删

### 📝 Deep Dive（每个 ⭐ 一张卡 / 一篇笔记）
- 模板见 `cards/_template.md`：
  - 一句话定义
  - 解决什么问题
  - 核心机制（3-5 步）
  - 最小代码
  - 我能用它做什么
  - 关联概念

### 🔗 Connect（归档时问 3 个问题）
1. **它解决哪一层的问题？**（主落点）
2. **它和哪几层有交互？**（跨层影响）
3. **我已知 vs 未知的边界？**（⭐📖🗑）
- 卡片 frontmatter 标 `主层: Lx`、`跨层: Ly, Lz`
- 关联已有卡片（双向链接）

### 🗣️ Teach（验证真懂）
- 写完卡片后，给 Ori 讲一遍
- Ori 用苏格拉底式追问逼真懂
- **能讲清楚 = 真懂**（Feynman 技巧）

### 📊 Audit（季度复盘）
- 用 framework 当 checklist
- 每层 0-3 分打分（0 没听过 / 1 听过说不出 / 2 能讲清楚 / 3 写过 demo）
- 找到薄弱层，下季度重点补
- 评估 framework 本身：结构还合理吗？需要新层吗？

## Cadence（频率表）

| 频率 | 做什么 | 时间 |
|---|---|---|
| **每天** | Capture | < 5 min |
| **周末** | Triage + Deep Dive (1-2 张卡) | 30-60 min |
| **每月** | Connect 整理（卡片之间关联补全） | 30 min |
| **每季度** | Audit（每层 0-3 分 + framework review） | 1 hr |
| **每年** | 框架大改（结构、新层、删除） | 2-3 hr |

## Ori 在 Loop 里的角色

| 步骤 | Ori 做什么 |
|---|---|
| Capture | 帮你快速归档到 inbox |
| Triage | 帮你判断 ⭐ vs 📖 vs 🗑 |
| Deep Dive | 帮你把文章提炼成卡片 |
| Connect | 帮你找新概念和已有卡片的关联 |
| Teach | **苏格拉底式追问**（核心） |
| Audit | 帮你做层打分 + 找薄弱环节 |

## 风险与应对

**最大风险**：loop 卡在某一步

| 卡点 | 表现 | 应对 |
|---|---|---|
| Capture 不持续 | inbox 空了好几天 | 强制每日 1 条（"不 capture 不准睡"） |
| Triage 累积 | inbox 堆了几十条 | 周六必 triage，否则下周堆积 |
| Deep Dive 拖延 | 标了 ⭐ 但没写卡 | 每张 ⭐ 设 7 天 deadline，过期降为 📖 |
| Connect 偷懒 | 卡片不标主层/跨层 | 模板强制（没有 frontmatter 不收） |
| Teach 跳过 | 写完不验证 | 写完自动进 Teach（Ori 主动追问） |
| Audit 漏做 | 季度没打分 | 季末 Ori 提醒 + 一起做 |
